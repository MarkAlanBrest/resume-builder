01_MASTER_GLOBAL_RULES
ENFORCEMENT NOTICE
All rules in this document are mandatory and override all other documents.
If a conflict exists, the rules in this document must be followed.

SECTION NUMBERING NOTICE
Section numbers are logical identifiers, not sequential ordering.
Missing section numbers are intentional and do not indicate omitted rules.

SECTION 1 — GLOBAL RESUME RULES (APPLY TO ALL PROGRAMS)
1.1 Resume Purpose
Generate clean, professional, employer-ready resume content tailored to the student’s program, job target, experience level, and strengths.
The resume content must be:
* Clear
* Concise
* Accurate
* Free of filler
* Free of exaggeration
* Free of assumptions
* Free of school advertising

1.2 Block-Based Resume Structure
Resume content must be generated and managed using structured content blocks.
Rendered resume blocks may include:
* Personal Information
* Professional Summary (include only when Section 4 conditions are met)
* Technical Skills
* Soft Skills
* Work Experience
* Military Experience (if applicable)
* Projects or Hands-On Experience (include only when Section 9 conditions are met)
* Education
* Certifications / Licenses
* Academic Metrics (optional; governed by Section 6 rules)
Optional blocks (include only if student provides content):
* Awards
* Volunteer Work
* Languages
If a block contains no valid content, it must be omitted entirely.

1.3 Separation of Responsibilities
The system operates using separation of concerns:
* AI is responsible for:
o Writing summaries
o Writing bullet points
o Organizing skills
o Adjusting emphasis and tone based on guidance
* The system is responsible for:
o Dates
o Credentials
o Certifications
o GPA and attendance
o Program selection
o Section inclusion logic
o Formatting and layout
AI must never format resumes or control layout.

1.4 Global Guidance Inputs
Some student inputs are provided as global guidance.
Global guidance:
* May influence tone, emphasis, and prioritization across any block
* Must never appear verbatim in the resume
* Must never be rendered as a visible resume section
Examples of global guidance include:
* Career objectives
* General comments or requests
* Emphasis instructions

SECTION 5 — SKILLS LOGIC
5.1 Skills Rules
* Always include a Skills section
* Separate Technical Skills and Soft Skills when appropriate
* Always include program-specific skills when applicable
* Include student-provided skills
* Include military-derived skills if applicable
* Do not invent skills
* Do not include soft skills unless they add value

SECTION 6 — GPA AND ATTENDANCE LOGIC
6.1 GPA Rules
Include GPA only if:
* GPA is 3.0 or higher
Format exactly as:
Academic Achievement: GPA 3.4

6.2 Attendance Rules
Include attendance only if:
* Attendance is 90% or higher
Use one of the following formats:
* “Maintained 92% attendance”
* “Achieved 92% attendance throughout the program”
Selection may vary between resumes.

SECTION 9 — PROJECTS / HANDS-ON EXPERIENCE LOGIC
Projects or hands-on experience may be included only when:
* The student provides project information
OR
* Section 16 program-based content support rules permit safe fallback
Rules:
* Projects must reflect training, labs, or verified exposure
* Do not exaggerate scope or responsibility
* Do not fabricate participation

SECTION 10 — OPTIONAL SECTIONS LOGIC
Optional sections may be included only if the student provides content:
* Awards
* Volunteer Work
* Languages
Never fabricate optional content.

SECTION 11 — PHYSICAL CAPABILITIES LOGIC
Include Physical Capabilities only if:
* The student explicitly provides physical capability information
OR
* The resume lacks sufficient content AND
* The program is physical in nature (e.g., Welding, Building Technology, CDL, Heavy Equipment, Diesel, Electrical)
Rules:
* Use only one physical capability if included
* Do not exaggerate
* Do not invent capabilities
* Do not use physical capabilities as filler

SECTION 13 — MISSING INFORMATION RULES
If a student leaves a field blank:
* Do NOT fabricate
* Do NOT guess
* Do NOT invent experience
* Do NOT add personal details
If a block contains no valid content, omit the entire block.

SECTION 14 — VARIATION RULES
To reduce repetitive resumes:
* Vary summary structure
* Vary bullet point openings
* Vary skill ordering
* Vary project descriptions
Variation rules:
* Are stylistic guidance only
* Must never override accuracy, clarity, or compliance

SECTION 15 — PLACEHOLDER & BLOCK MAPPING
Templates must map directly to structured blocks and fields.
Rules:
* Do not flatten blocks into single strings
* Do not render empty fields
* If a block or field cannot be populated, omit it entirely
* Conditional rendering must remove the entire line or section

SECTION 17 — FINAL RULES FOR ALL PROGRAMS
17.1 No Fabrication Rule
The system must never invent:
* Experience
* Certifications
* Licenses
* Awards
* Volunteer work
* Languages
* Tools the student did not use

17.2 No School Advertising Rule
The resume must never include:
* School mission statements
* Promotional language
* Instructor names
* Course numbers
* Campus marketing descriptions

17.3 No Personal Information Rule
Do not include:
* Age
* Gender
* Marital status
* Photos
* Social Security numbers
* Driver’s license numbers (except CDL)

17.4 No References Section
Do not include:
* References
* “References available upon request”

17.5 Resume Output Standards
The final resume must be:
* Clean
* Professional
* Single-column
* ATS-friendly
* Free of decorative elements (icons, colors, graphics, tables)
Simple horizontal lines are permitted.

END OF MASTER GLOBAL RULES
